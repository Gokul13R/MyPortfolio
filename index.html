Cross-Platform Game Master Controller

I developed a custom GameMasterController script to manage cross-platform configuration within the project. Instead of manually adjusting input systems, UI layouts, or advertisement settings for each build target (PC, Android, iOS, WebGL), the entire setup can be switched using a single dropdown in the Unity Inspector.

The controller provides:

  • Platform-specific input handling, utilizing Unity’s New Input System, which makes adding support for new platforms straightforward and scalable.
  • Automatic UI scaling and layout adjustments based on device type and screen resolution.
  • Integrated advertisement control to enable or disable ads depending on the game version.

This enables efficient release management:

  • Free Version (with ads) → Advertisements are automatically enabled.
  • Premium Version (ad-free) → All ads can be disabled with a single toggle.

By centralizing platform configuration and leveraging the new Input System, the project becomes modular, easy to maintain, and quick to deploy across multiple platforms — with the flexibility to expand to additional platforms in the future without rewriting core systems.

Modular Enemy Wave & Difficulty Scaling System

This system uses a ScriptableObject to manage enemy wave spawning in the game. The ScriptableObject stores a list of enemy prefabs along with their attributes such as health, damage, movement speed, and behavior type.

Each wave is fully configurable, including:

  • The total number of enemies to spawn
  • Which enemy types appear in that wave
  • The difficulty scaling (power level percentages for stats)
  • AI behavior settings (e.g., chase the player or patrol)

By centralizing this data in a ScriptableObject, new enemies or wave variations can be added without modifying code. This allows fast balancing and iteration during game design. Changing enemy stats or adding new enemy types can be done in a few clicks inside the Unity inspector.

This approach makes the enemy wave system flexible, scalable, and easy to maintain for future updates or difficulty adjustments.

Modular Enemy AI Data & Attack Configuration System

This system uses a ScriptableObject to store and manage all AI enemy data. Each enemy has its own data asset that defines core attributes such as health, movement speed, chase behavior, animation references, and sound effects.

The ScriptableObject also contains a configurable attack list, allowing each enemy to have multiple unique attacks.
For every attack, the designer can adjust:

  • Damage value
  • Time between attacks
  • Animation clip reference
  • Attack direction (used for camera shake orientation)
  • Attack power level (used to scale camera shake intensity)
  • Individual sound effects per attack

Because all attack data is stored in an array inside the ScriptableObject, new attack types can be added without writing additional code. This makes it easy to create diverse enemy behaviors and animations.

This enemy data system integrates with the Modular Enemy Wave & Difficulty Scaling System, which scales stats such as health, damage, and power levels by percentage depending on the wave progression. This ensures consistent difficulty progression while keeping the system easy to balance and expand.

Gun Data System (Scriptable Object)

I developed a ScriptableObject-driven gun data system to manage all weapon stats, visual effects, and sound effects in a modular way. Each gun has its own ScriptableObject that stores:

  • Weapon Identity: Gun name, type, and UI icon for weapon switching
  • Combat Stats: Damage, rate of fire, bullet speed, recoil, magazine size, and reload speed
  • Spread & Recoil Control: All recoil behavior and bullet spread are handled entirely through code (no animation-based recoil). This allows consistent and predictable gun handling.
  • Audio Integration: Each weapon asset includes its firing, reload start, and reload end sounds, along with individual volume controls for fine-tuning.

The gun stats can be dynamically modified based on power-up cards the player receives after each wave. For example, if the player selects a “+50% Damage” card, the damage value in the gun data is updated at runtime without changing the original ScriptableObject asset.
This allows:

  • Flexible weapon upgrades
  • Easy balancing and tuning in the Inspector
  • Clean separation of data from logic
  • Reusable weapons and upgrade behaviors across different guns

Overall, this system makes adding or adjusting weapons scalable and efficient, while also supporting the game’s wave-based upgrade mechanics.

Centralized Audio Manager System

I implemented a centralized Audio Manager that allows playing sound effects from anywhere in the project with a single function call:

AudioManager.Instance.PlaySoundEffectClip(SoundFX, transform,SoundFXVol);

This system spawns and destroys temporary audio sources at runtime, which means individual GameObjects (like enemies, weapons, UI buttons, or environment elements) do not need their own AudioSource components. This keeps the scene hierarchy clean, reduces component overhead, and makes managing sound levels and mixers much simpler. All sound groups (Music, UI, EnemyFX, etc.) are routed through a unified Audio Mixer, allowing full volume control from the in-game settings menu.

Performance & Optimization

To ensure the game runs smoothly on low-end mobile devices, I applied several optimization techniques focused on reducing rendering cost and draw calls:

  • Selective Occlusion Culling: Instead of enabling occlusion culling for every object, I applied it only to high-polygon environment models. This reduced overdraw and saved memory, without increasing the final build size unnecessarily.
  • Unlit Shader Usage: All environment and neon elements use Unlit shaders, which removes real-time lighting and reflection calculations. This significantly lowers GPU workload and provides a clean, stylized visual look suitable for mobile performance.
  • Texture Atlas: I combined multiple textures into a single 512×512 texture atlas, minimizing material and texture switches. This reduces draw calls and improves batching efficiency.

With these optimizations, the game consistently achieves 165+ FPS in the Unity Editor on a Ryzen 3 3200G with 16GB RAM and no dedicated GPU, demonstrating strong performance scalability for mobile hardware.

Key Features

Fully Solo Developed

Designed and built end-to-end by me, including gameplay programming, system architecture, UI, level logic, and all 3D modeling, texturing, and animations.

Endless Procedural Environment

The ground path, obstacles, trees, rocks, ramps, and stars are procedurally generated. The game creates new segments ahead and removes the old ones behind the player for optimized performance and seamless endless gameplay.

All Assets Made From Scratch

All environment models and character models were created from scratch in Blender, giving the game its own unique visual style.

Binary Data Saving

Player data such as high score, total stars, and game settings are saved using binary serialization for efficient, lightweight, and secure data storage.

Multiple Touch Control Modes

Includes two different input control layouts so players can choose their preferred control style.

Adjustable Camera Views

Offers multiple camera modes such as Follow Cam and Hard Look mode, allowing players to customize the gameplay feel.

Monetization

Integrated ads to support in-game monetization while keeping the gameplay experience smooth.
